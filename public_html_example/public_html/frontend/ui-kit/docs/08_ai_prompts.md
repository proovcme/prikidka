# Промты для ИИ-верстальщика

Этот файл — инструкция одновременно для человека и для ИИ. Скопируй нужный промт и скорми вместе с брендбуком.

## Перенести брендбук на существующий сайт

Я прикладываю файл OVCuikit.html — это интерактивный брендбук сайта ovc.me.
В нём описаны: цветовые токены (CSS-переменные), типографика, компоненты —
карточки, кнопки, хедер, футер, формы, таблицы, плитки статей.

Дополнительно реализовано:
- Переключатель тёмной/светлой темы в хедере (кнопка "theme --light/--dark"),
  состояние сохраняется в localStorage. Класс body.light-theme переопределяет токены.
- OVC Trigger скрипт в секции 11 — интерактивная расшифровка домена.

Твоя задача: перенести визуальный стиль из брендбука на мой сайт [URL или код].
Используй те же CSS-переменные, шрифтовой стек (--font-code, --font-ui),
цвета из :root и паттерны компонентов. Не меняй контент, только стиль.

## Создать новую страницу статьи

Используй брендбук OVCuikit.html как дизайн-систему.
Создай HTML-страницу статьи для подсайта [rag/toc/bim].ovc.me.

Требования:
- Хедер и футер из секции 02 брендбука (тёмная тема)
- Тема статьи: .theme-article-[rag/toc/bim]
- Структура: article-prompt → article-title → article-content
- Используй callout-блоки [INFO], [WARN], [OK] где уместно
- Плитки-переходы к связанным статьям в конце (секция 03b)
- OVC Trigger скрипт из секции 11 в хедере

## Добавить OVC Trigger на существующий сайт

В секции 11 файла OVCuikit.html есть скрипт OVC Trigger.
Добавь его на мой сайт [URL или код].

Логика: клик на "ovc.me" в хедере меняет расшифровку.
85% — серьёзные варианты, 15% — пасхалки, на 10-м клике — мета-пасхалка.
Стилизуй под цвета моего сайта, сохрани механику без изменений.

## Сгенерировать новые расшифровки OVC

Придумай 5 новых расшифровок для домена ovc.me по формуле [O][V][C] : [M][E].

Правила:
- Строго инженерная / управленческая терминология
- Без личных данных и неформальных фраз
- Стек на выбор: RAG (векторы, контекст, ML), TOC (цепочки, узкие места),
  BIM (виртуальное строительство, объекты)
- Язык: английский или русский (О-В-Ц : М-Э/Е)
- Пасхалки: технично серьёзные по форме, но с иронией по смыслу для тех кто в теме

Пример серьёзной: "Optimized Value Chain : Management Execution"
Пример пасхалки: "Object Vanished Completely : Model Exists"
