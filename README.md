# ПИР-Калькулятор онлайн | Оценка стоимости и планирование проектирования по ТОС

Профессиональный веб-инструмент для автоматизации расчёта стоимости проектно-изыскательских работ (ПИР) и календарного планирования на основе двух подходов: **классического затратного метода** и **Теории ограничений систем (TOC / ТОС)**.

Репозиторий проекта: [https://github.com/proovcme/prikidka](https://github.com/proovcme/prikidka)

---

## 🚀 Основные возможности

* **Два режима планирования (Простой / ТОС):**
  * **Простой (Смета):** Быстрый расчет трудозатрат, стоимости и состава работ по СБЦ без учета рисков и неопределенностей.
  * **Продвинутый (ТОС):** Полноценное планирование с расчетом единого защитного временного буфера проекта на основе матрицы рисков.
* **Интерактивная диаграмма Ганта:**
  * Визуализация задач на базе библиотеки `Frappe Gantt`.
  * Динамическое каскадирование дат при изменении длительности или связей (предшественников).
  * Назначение нескольких исполнителей на задачу с указанием процента их загрузки.
  * Автоматический расчет ФОТ задачи на основе задействованных ресурсов и их ставок.
* **Гибкая ролевая модель (Директор / ГИП):**
  * **Директор (`?role=director`):** Полный доступ ко всей финансовой структуре проекта (ставки, ФОТ, накладные, CAPEX, рентабельность, налоги, итоговые финансовые показатели).
  * **ГИП (`?role=gip`):** Конфиденциальный режим для Главного инженера проекта. Полностью скрыты все финансовые поля и таблицы ставок. Доступно только управление составом работ, сроками, рисками, Гантом и экспортом.
* **Матрица рисков и Финансовый резерв:**
  * Оценка вероятности (1–3) и влияния (1–3) рисков с цветовым кодированием зон угрозы.
  * Расчет денежного резерва по методу ожидаемой стоимости (EMV): `Резерв = Ущерб × Вероятность`.
  * Расчет ТОС-буфера времени: `Буфер = Задержка × Вероятность` для рисков желтой и красной зон.
* **Амортизация CAPEX в составе проекта:**
  * Учет капитальных затрат (лицензии ПО, серверы, оборудование).
  * Автоматический расчет линейной амортизации оборудования пропорционально длительности проекта для включения в себестоимость.
* **Автосохранение и Синхронизация:**
  * Локальное автосохранение всех внесенных данных в `LocalStorage` браузера.
  * Функция сохранения проекта в облачную базу данных (кнопка «💾 В облако») и загрузка по ID.

## 🛠 Технологический стек

* **Frontend:** Чистый HTML5, Vanilla CSS3 (современная адаптивная верстка, темная тема, стеклянный морфизм, плавные CSS-анимации), Vanilla JavaScript (модульная архитектура ES6).
* **Библиотеки:** `Frappe Gantt` (интерактивный календарный график).
* **Backend:** PHP (маршрутизация, облачное сохранение проектов в JSON-хранилище, серверные пресеты CAPEX).

## 📂 Структура файлов проекта

* 📄 `calc.php` — Главный файл приложения (структура интерфейса, стили, разметка модальных окон инструкций).
* 📄 `app.js` — Основной контроллер приложения (инициализация, роутинг ролей, локальное сохранение, переключение режимов).
* 📄 `economics.js` — Логика расчета ФОТ, управления списком ролей исполнителей и их дневными ставками.
* 📄 `risks.js` — Логика матрицы рисков, расчет EMV финансового резерва и ТОС-буфера времени.
* 📄 `gantt.js` — Интеграция с Frappe Gantt, логика связей задач (предшественников), каскадирование дат и пересчет ФОТ задач в реальном времени.
* 📄 `csv-manager.js` — Импорт и экспорт разделов проектирования в формате CSV.
* 📄 `api_projects.php` — Серверный API для сохранения и загрузки проектов.

## 📖 Подробное руководство по разделам

### Раздел 01: Реквизиты проекта
Ввод метаданных проекта: шифр, название объекта, заказчик, плановые даты и статус готовности.

### Раздел 01.3: Ставки специалистов (Роли)
Таблица базовых ставок специалистов (₽/чел-день). Доступно добавление новых ролей (например, *Сметчик — 6000 ₽*) и редактирование существующих. Любые изменения мгновенно синхронизируются с чекбоксами исполнителей на диаграмме Ганта.

### Раздел 01.5: Комплектность исходных данных
Чек-лист входящей документации (ГПЗУ, ИГИ, ИГДИ, ТУ, EIR и др.). Неотмеченные пункты сигнализируют о рисках задержки старта проекта.

### Раздел 02: Состав работ и Экспертиза
Выбор стадии (П, Р, Изыскания, ЭС, ТИМ), автоматический расчет стоимости прохождения экспертизы (ГГЭ, Регион, Частная), выбор разделов из СБЦ или добавление собственных. По клику на кнопку «Перенести в график» разделы автоматически экспортируются в календарный план.

### Раздел 03: Матрица рисков (Режим ТОС)
Матричная оценка неопределенностей проекта. Риски с рейтингом вероятности и влияния $\ge 4$ автоматически формируют:
1. **Финансовый резерв:** Добавляется в итоговую смету (Раздел 07).
2. **ТОС-буфер времени:** Пристраивается защитным блоком в конец графика Ганта.

### Раздел 04: Календарный график и Гант
Интерактивное планирование. Позволяет связывать задачи через поле «Предшественник» (например, связь `1` означает, что задача начнется строго после завершения задачи №1). Назначение исполнителей рассчитывает ФОТ каждой задачи на лету. В режиме ТОС в конце графика выводится единый буфер, защищающий критический путь проекта от опозданий.

### Раздел 05: Капитальные затраты (CAPEX)
Учет закупки специализированного софта (Revit, AutoCAD, ПК Лира и др.) или серверов. Расчет линейного износа за время работы над проектом предотвращает кассовые разрывы на обновление ПО.

### Раздел 06 & 07: Коэффициенты сложности и Финансовый итог
Настройка накладных расходов, коэффициентов (срочность, удаленность, сложность), нормы прибыли компании и налоговых отчислений. Раздел 07 сводит все показатели в единый финансовый результат с графиком платежей (аванс / расчет).

## 🚀 Установка и Быстрый запуск

### 💻 1. Локальный запуск (быстрый старт без СУБД)
Для локального ознакомления и тестирования калькулятора база данных не требуется (все данные автосохраняются в `LocalStorage` браузера).
1. Убедитесь, что у вас установлен PHP.
2. Склонируйте репозиторий и перейдите в его папку:
   ```bash
   git clone https://github.com/proovcme/prikidka.git
   cd prikidka
   ```
3. Запустите локальный PHP-сервер:
   ```bash
   php -S 127.0.0.1:8123
   ```
4. Откройте проект в браузере:
   * **Режим Директора:** `http://127.0.0.1:8123/calc.php?role=director`
   * **Режим ГИПа:** `http://127.0.0.1:8123/calc.php?role=gip`

---

### 💾 2. Развертывание на сервере (Деплой с Базой Данных)
Для работы функции сохранения проектов в облаке («💾 В облако» и загрузка по ID) необходимо настроить СУБД MySQL / MariaDB.

#### Шаг A: Создание базы данных
1. Создайте пустую базу данных в панели вашего хостинга или через терминал (например, `prikidka_db`).
2. Импортируйте структуру таблиц из файла `database.sql`:
   ```bash
   mysql -u пользователь -p prikidka_db < database.sql
   ```
3. *(Опционально)* Импортируйте тестовые данные из файла `seed.sql` для предзаполнения базы:
   ```bash
   mysql -u пользователь -p prikidka_db < seed.sql
   ```

#### Шаг B: Настройка конфигурационных файлов
1. Скопируйте файл-шаблон настроек подключения `db_config.example.php` и переименуйте его в `db_config.php`:
   ```bash
   cp db_config.example.php db_config.php
   ```
2. Откройте созданный `db_config.php` и впишите реквизиты вашей базы данных:
   ```php
   <?php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'prikidka_db');
   define('DB_USER', 'ваш_пользователь_бд');
   define('DB_PASS', 'ваш_пароль_бд');
   ```
3. Скопируйте основной файл подключения `db.example.php` в `db.php`:
   ```bash
   cp db.example.php db.php
   ```

#### Шаг C: Настройка прав и папок
Убедитесь, что веб-сервер имеет права на запись в папку проекта для временного кэширования или сохранения JSON-конфигураций (если они включены). На большинстве серверов достаточно прав `755` для папок и `644` для PHP-файлов.
